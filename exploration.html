After-sale products management platform

Web Design

C#

Database

Ajax

Multi-Users

ASP.NET

This is a website for managing after-sales products, which I completed during my second internship. Many experienced and excellent colleagues guided me in realizing this work.

It is built on ASP.NET, with the interface implemented through web pages. The backend is written in C#, and the data is stored in the enterprise server's database.

The purpose of this platform is to manage instruments that users send back for repairs. Staff in various positions are shown different information to manage and control the registration, evaluation, repair, and shipping statuses of the instruments, ultimately returning them to the customers.

Login page of the platform

• Structure of the interface •

Different tables list instruments in various states, and individuals in different positions can access the corresponding tables. Managers can oversee the entire process.
Every User can change the state of the instrument by click the buttons.

The core of the platform is a database system. When users modify or add data, the changes are presented in real-time on the web page through AJAX technology.

*Schema generated by Microsoft Sql Server Management Studio

I try to introduce briefly, thus many pages in the platform have been omitted.

This platform has undergone multiple revisions and tests, and is now officially in use.

Instrument Control Software Development

This is software for displaying instrument measurement data on a computer and recording it. This software can also control the instrument and communicate with it.

After the measurements are completed, users can download the data in Excel spreadsheet format.

This software was completed by me during my internship, and it is a desktop program developed using Visual Studio .NET framework.

*Display of Instrument and the software

*Main page of the software

*Adjustment of instrument parameters

The software displays real-time instrument measurement data on the left side.

The right side of the software is used to display recorded data.

Users can modify the instrument parameters to obtain different data.

This project has helped me gain a better understanding of precision instruments and communication.

The software utilizes a communication library provided by the company.

This software has been officially adopted by the company after several modifications.

"To be the one plays game, or the one designs game?"
Mother used to tell me when I was young.
From a moment I realized that, creating and designing games, that's the meaningful thing to do.

For years I've been learning how to use Unity Engine to design games, try to design the cute and beautiful characters and let users to control them.

Here I would like to show a 2D Game I've designed, called "Piggy Happy". I've drew 5 characters by Adobe Illustrator, and created skeletal animations for them. The code portion is implemented in C#.

Use Adobe Illustrator to create every part of the characters.

In Unity Engine, bind all parts of the chracter by skeleton.

With C#, create action functions for a character to apply force and velocity when the character needs to move and jump.

Read and process input for two players' actions, and trigger character action functions to make the characters move.

When characters are moving or jumping, add corresponding animation effects.
